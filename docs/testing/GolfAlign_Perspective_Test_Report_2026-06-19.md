## 2. 기획자 관점 테스트

### 통과

- 첫 화면에서 브랜드 문구 `스윙을 기록하고, 성장을 정렬하다`가 명확히 보인다.
- 로그인 전 화면에서 회원가입, 로그인, Google 계정 시작 흐름이 보인다.
- 회원 모드 홈에서 업로드, 새 피드백, 오늘 할 훈련, 최근 기록이 한 화면 흐름으로 연결된다.
- 라운드 모집은 `개발 중`으로 표시되어 MVP 핵심 흐름을 크게 방해하지 않는다.
- 마이 화면에 언어 설정, 저장 공간 정책 안내가 있어 글로벌/저장 정책 확장 위치가 보인다.
- 업로드 화면에서 원본 영상은 기기에 두고 썸네일/캡쳐/피드백 데이터 중심으로 간다는 방향이 설명된다.

### 운영 정책으로 확인

- 로그인 첫 화면의 아이디 저장은 사용자 편의 기능으로 본다. 단, 사용하는 기기에만 저장되도록 유지한다.
- 관리자 미리보기 버튼은 개발/관리자 확인용이며 실제 사용자에게는 노출하지 않는 전제로 본다.

### 보완

- 실제 가입/로그인/저장 플로우는 아직 Google Sheets 서비스 계정 환경변수 등록 전이라 완결 검증이 되지 않았다.

### 차단

- 회원가입 후 DB 저장, 프로방 생성, 메시지 저장, 기록 저장은 현재 프로덕션 환경에서 차단된다.
- 원인: Vercel Production에 `GOOGLE_CLIENT_EMAIL`, `GOOGLE_PRIVATE_KEY`가 없어 로컬 JSON DB fallback이 발생하고, Vercel의 읽기 전용 파일시스템 때문에 `/api/users`가 500을 반환한다.

## 3. 일반회원 관점 테스트

### 홈

결과: 통과

- 오늘 할 훈련이 `어프로치 100개` 형태로 바로 보인다.
- 최근 기록이 영상/이미지, 피드백 완료/확인 필요 상태로 구분된다.
- 새 피드백 카드가 목표 행동인 `스냅샷에서 확인`으로 이어진다.
- 영상 5초 권장, 최대 7초 안내가 홈 업로드 버튼 아래에 보인다.

### 기록

결과: 보완

- `내 기록` 화면에서 프로 피드백 완료, 프로 확인 대기, 개인 기록 영역이 구분된다.
- 공유 기록과 개인 기록이 분리되어 보인다.
- 보완: 버튼 접근성 이름에 긴 카드 전체 텍스트가 들어가 자동화/스크린리더 기준으로 다소 무겁다.
- 보완: `기록`이라는 이름이 하단 탭과 카드/섹션에 중복되어 자동화 기준으로 구분이 필요했다.

### 업로드

결과: 통과

- 업로드 화면에서 영상/이미지 선택 영역이 명확하다.
- `5초 권장`, `최대 7초까지 저장`, `느린 재생`, `캡쳐 피드백`이 짧은 문구로 보인다.
- 기록 종류가 `스윙`, `퍼팅`, `어프로치`로 구분된다.
- 자세 분석 포인트가 `발 정렬`, `팔/손목`, `고개/시선`, `허리/골반`, `등/상체`, `무릎/체중`으로 보인다.
- `업로드 저장 시 가입한 프로방에 공유` 체크박스가 보인다.
- 개인 기록과 프로방 공유의 관계 설명이 있다.

### 훈련

결과: 통과

- 훈련 과제가 투두리스트처럼 보인다.
- 전체, 프로 추천, 개인 훈련, 완료 필터가 보인다.
- 방 공통 드릴과 개인 훈련이 구분된다.
- 결과 기록 버튼이 있어 훈련 완료 후 결과 저장 흐름이 보인다.

### 메시지

결과: 보완

- 프로 찾기 검색창이 보인다.
- 프로 목록과 신청 버튼이 보인다.
- 내 연결과 메시지함이 보인다.
- 보완: `신청` 버튼이 여러 개라 어떤 프로에게 신청하는지 버튼 자체만으로는 구분이 약하다.
- 보완: 개인 메시지의 수신자 선택은 종이비행기 버튼 안내가 있지만, 현재 추출된 화면 기준으로 종이비행기 버튼 텍스트가 명확히 보이지 않는다.

### 마이

결과: 통과

- 회원 프로필 저장 영역이 보인다.
- 언어 설정에 `한국어`, `English`가 보인다.
- 저장 공간 안내가 보인다.
- 영상은 원본 다운로드/기기 재생, 썸네일/스냅샷 중심이라는 정책 설명이 보인다.

## 4. 프로 관점 테스트

### 프로 홈

결과: 통과

- 피드백 대기 기록 목록이 보인다.
- 내 레슨방 요약과 초대코드가 보인다.
- 프로 프로필 저장 영역이 보인다.

### 레슨방

### 피드백

- 프로 홈의 피드백 대기 기록에서 피드백 작성 화면으로 진입할 수 있었다.
- 스냅샷 피드백 화면에 다음 도구가 보인다:
  - 원본
  - 발 정렬
  - 팔/손목
  - 고개/허리
  - 뼈각도 켜짐
  - 지우개
  - 복원
  - 이동
  - 텍스트
  - 선
  - 화살표
  - 사각
  - 원
  - 삭제
- 발/팔/고개/허리/등 각도 배지가 보인다.
- 목표, 신경쓸 것, 스냅샷 말풍선, 다음 추천 입력이 보인다.
- 보완: 현재 확인한 피드백 샘플은 이미지 기록이라 `video` 재생 요소와 0.5배속 실제 동작은 확인하지 못했다.
- 보완: 본/각도 일부 지우기와 텍스트 자유 이동은 화면 도구가 보이지만 실제 조작 저장까지는 별도 테스트가 필요하다.

### 훈련 확인

결과: 통과

- 회원의 훈련 결과가 제출됨 상태로 보인다.
- 프로 코멘트 입력 영역이 있다.
- `확인 완료` 버튼이 있어 운영형 피드백 흐름이 보인다.
- 반복 과제에 대해 짧은 코멘트를 남기는 기획 방향과 맞다.

### 메시지

결과: 보완

- 회원 찾기 검색창이 보인다.
- 회원 목록, 초대 버튼, 내 회원, 메시지함이 보인다.
- 프로방 신청과 훈련 결과 메시지가 구분되어 보인다.
- 보완: `초대` 버튼이 여러 개라 버튼 자체 accessible name만으로 대상 회원 구분이 약하다.
- 보완: 신청 승인 후 즉시 갱신은 DB env 등록 후 실제 쓰기 테스트가 필요하다.

## 6. 재테스트 필요 항목

Google Sheets 서비스 계정 환경변수 등록 후 반드시 다시 확인해야 한다.

- 회원가입 후 Google Sheets `users` 반영
- 프로가입 후 `users`, `pro_profiles` 반영
- 로그인 후 역할별 화면 진입
- 프로방 생성 후 `lesson_rooms`, `room_invites` 반영
- 회원 신청 후 `room_join_requests`, `messages` 반영
- 프로 승인 후 `room_members`, `messages` 반영
- 개인 메시지 송수신 후 `messages` 반영
- 훈련 과제 생성 후 `training_assignments` 반영
- 훈련 결과 제출 후 `training_results` 반영
- 영상 기록 저장 후 `records`, `shared_records` 반영
- 영상 기록의 `archive_policy`, `archive_after_days`, `archive_due_at`, `sync_status=archive_pending` 반영
- 이미지 기록의 `archive_policy=none` 반영

## 7. 우선 수정 제안

1. Vercel Production 환경변수에 Google Sheets 서비스 계정 값을 등록한다.
2. 카드형 버튼의 accessible name을 짧게 정리한다.
3. 메시지 목록의 `신청`, `초대` 버튼에 대상 이름을 포함한다.
   - 예: `김프로에게 신청`, `박회원 초대`
4. 실제 영상 기록 샘플로 0.5배속, 캡쳐, 뼈각도 레이어, 지우개, 텍스트 이동을 별도 재검증한다.
5. Google Sheets 연결 후 실제 회원가입, 프로가입, 프로방, 메시지, 기록 저장 흐름을 다시 테스트한다.
