# GolfAlign 기획자/사용자/프로 기준 테스트 체크리스트 v0.1

목적: 개발자 관점의 API/DB 테스트와 별도로, 실제 서비스 사용 흐름이 이해하기 쉽고 GolfAlign의 기획 의도에 맞는지 확인한다.

## 1. 기획자 관점 테스트

### 서비스 이해도

- 첫 화면에서 GolfAlign이 어떤 앱인지 5초 안에 이해되는가?
- "스윙을 기록하고, 성장을 정렬하다"라는 브랜드 메시지가 화면 흐름과 맞는가?
- 회원/프로/관리자 역할이 섞여 보이지 않는가?
- 프로방, 개인 기록, 훈련 과제, 피드백의 관계가 자연스럽게 이어지는가?
- 영상 원본은 무겁게 저장하지 않고 썸네일/스냅샷 중심으로 간다는 방향이 화면에서 과하게 복잡하지 않은가?

### MVP 범위 적합성

- 현재 기능이 너무 소셜앱처럼 커 보이지 않는가?
- 라운드 모집/친구목록 등 보류 기능이 MVP 핵심을 방해하지 않는가?
- 프로방 연결 기능이 가입률/유지율을 높일 만한 구조로 보이는가?
- 일반회원이 프로방에 없어도 개인 기록/훈련을 쓸 수 있는가?
- 프로방 가입 후 개인 기록을 프로에게 공유하는 흐름이 명확한가?

### 화면 우선순위

- 홈에서 가장 중요한 행동이 분명한가?
- 기록 업로드, 피드백 확인, 훈련 확인이 하단 탭에서 쉽게 찾을 수 있는가?
- 메시지 탭이 프로방 신청/초대/개인메시지를 담기에 충분한가?
- 프로 연결 정보가 너무 길게 보이지 않고 한 줄 요약 후 상세 진입 구조로 보이는가?
- 관리자 미리보기/개발용 버튼이 실제 사용자에게 노출되지 않을 구조인가?

### 데이터/운영 기준

- 회원가입 정보가 Google Sheets DB로 저장되는 흐름이 준비되어 있는가?
- 프로필 사진은 Drive 저장, 장기 미접속자는 NAS 이동 후보라는 정책이 명확한가?
- 영상 원본은 피드백 완료 후 7일, 최대 업로드 후 14일 NAS 이동 정책과 맞는가?
- 관리자 화면에서 회원 수, 프로 수, 저장 용량, 오류 상태를 볼 필요가 있는가?
- 정식 출시 전 Google OAuth, Google Sheets, NAS 연결 과제가 누락되지 않았는가?

## 2. 일반회원 관점 테스트

### 가입/로그인

- 회원가입에서 아이디/비밀번호 입력이 어렵지 않은가?
- 회원/프로 선택에서 일반회원이 헷갈리지 않는가?
- 로그인 시 이전에 입력한 아이디가 노출되어도 보안상 문제가 없는 방식인가?
- 비밀번호만 입력하고 재로그인하는 흐름이 편한가?
- 15분 무조작 후 자동 로그아웃이 자연스럽게 안내되는가?

### 홈

- 오늘 해야 할 훈련이 바로 보이는가?
- 개인 기록과 프로가 준 과제가 구분되는가?
- 최근 피드백을 쉽게 찾을 수 있는가?
- 프로방 가입 여부와 상관없이 개인 기능을 쓸 수 있는가?
- 글자가 작거나 설명이 너무 길지 않은가?

### 기록 업로드

- 영상/이미지 업로드 버튼을 쉽게 찾을 수 있는가?
- 영상은 5초 권장, 7초 허용 안내가 부담스럽지 않은가?
- 촬영 각도가 아니라 몸의 각도/자세 분석이라는 의미가 명확한가?
- 기록 종류와 몸 각도 선택이 잘 되는가?
- 업로드 중 프로방 공유 체크가 이해되는가?
- 프로방 공유를 체크하면 개인 기록과 프로방 공유 기록이 모두 남는다고 이해되는가?
- 업로드 후 썸네일/스냅샷이 어디에 저장됐는지 사용자가 알 수 있는가?

### 기록 확인

- 내가 올린 영상/이미지 기록 목록을 쉽게 찾을 수 있는가?
- 썸네일 중심으로 기록을 빠르게 구분할 수 있는가?
- 영상은 다운로드/기기 재생 방식이라는 점이 자연스러운가?
- 스냅샷을 눌렀을 때 피드백 대상 이미지로 이해되는가?
- 내 기록을 프로방에 나중에 공유할 수 있는가?

### 프로방/메시지

- 프로 찾기 검색이 쉬운가?
- 프로방 신청 버튼이 명확한가?
- 프로에게 받은 초대 메시지를 이해하기 쉬운가?
- 초대 수락 후 내 연결/내 프로방이 즉시 갱신되는가?
- 개인메시지와 초대/신청 메시지가 헷갈리지 않는가?

### 훈련

- 훈련 과제가 투두리스트처럼 보이는가?
- 완료 체크가 잘 되는가?
- 훈련 결과에 영상/이미지를 선택적으로 첨부할 수 있는가?
- 프로가 준 훈련과 내가 직접 만든 훈련이 구분되는가?
- 완료 후 결과가 기록으로 남는다는 점이 이해되는가?

### 피드백

- 프로 피드백이 목표와 신경 쓸 점 중심으로 짧게 보이는가?
- 스냅샷 위 스티커 메모가 화면을 많이 가리지 않는가?
- 자세 본/각도 표시를 켜고 끄는 기능이 이해되는가?
- 텍스트 색상이나 도형이 배경 위에서도 잘 보이는가?
- 피드백을 보고 다음 훈련으로 이어질 수 있는가?

## 3. 프로 관점 테스트

### 가입/첫 사용

- 프로 가입 시 일반회원과 다르게 전문 관리 앱이라는 느낌이 드는가?
- 프로필/소속/레슨 영역 입력이 과하지 않은가?
- 첫 로그인 후 바로 프로방을 만들 수 있는가?
- 프로방 초대 링크를 카카오톡/문자로 보내기 쉬운가?
- 프로가 앱을 통해 체계적으로 관리한다는 인상을 줄 수 있는가?

### 프로방 관리

- 프로방 생성이 쉽고 빠른가?
- 방 이름, 목적, 초대 코드가 이해되는가?
- 회원 목록에서 이름과 프로필 사진 정도만 보여 개인정보 부담이 적은가?
- 특정 회원 세부 기록은 개인별로만 들어가서 볼 수 있는가?
- 회원이 많아져도 목록 스크롤이 자연스러운가?

### 회원 신청/초대

- 회원 검색이 쉬운가?
- 프로가 회원을 찾아 초대할 수 있는가?
- 회원이 신청한 내용을 프로가 확인하고 원하는 방에 넣을 수 있는가?
- 승인 후 내 회원 목록이 즉시 갱신되는가?
- 메시지함에서 신청/초대/개인메시지가 과하게 복잡하지 않은가?

### 훈련 과제

- 프로가 드릴/훈련 추천 목록을 쉽게 선택할 수 있는가?
- 직접 입력으로 과제를 추가할 수 있는가?
- 개인 과제와 방 공통 과제를 구분할 수 있는가?
- 과제에 목표, 횟수, 체크 방식, 영상/이미지 첨부 여부를 설정할 수 있는가?
- 회원 결과를 보고 다음 과제를 주는 흐름이 자연스러운가?

### 영상/스냅샷 피드백

- 회원이 올린 영상/이미지를 쉽게 찾을 수 있는가?
- 영상 0.5배속 재생이 프로의 분석 목적에 맞는가?
- 원하는 순간을 캡쳐해 스냅샷 피드백을 남길 수 있는가?
- 캡쳐 이미지 위에 본/각도 표시를 켜고 끌 수 있는가?
- 본/각도 일부를 지우개로 지울 수 있어야 하는가?
- 선, 도형, 텍스트, 색상 선택이 피드백 작성에 충분한가?
- 텍스트 메모를 자유롭게 이동할 수 있는가?
- 최종 피드백은 짧은 목표/주의점 중심으로 회원에게 보이는가?

### 수업 운영

- 어프로치 100개, 스윙 100개처럼 단순 반복 과제에 짧은 피드백을 남길 수 있는가?
- "이번 주는 충분하니 다음 주 다른 과제 추천" 같은 운영형 피드백이 가능한가?
- 회원별 훈련 완료 여부를 빠르게 볼 수 있는가?
- 프로가 회원에게 추천 훈련을 쉽게 전송할 수 있는가?
- 프로가 피드백 내역과 성장 기록을 비교하기 쉬운가?

## 4. 공통 사용성 테스트

### 가독성

- 주요 글자 크기가 40대 이상 골프 사용자에게 충분한가?
- 버튼 크기가 손가락 터치에 충분한가?
- 노랑/주황 대표색이 너무 튀지 않고 활동적으로 보이는가?
- 카드와 배지가 깔끔하게 구분되는가?
- 긴 설명 대신 짧은 문장과 상태 배지가 중심인가?

### 모바일 사용성

- 하단 탭이 손가락으로 누르기 쉬운가?
- 업로드/기록/피드백 화면이 오른쪽으로 밀리지 않는가?
- 영상/이미지 뷰어가 화면을 넓게 쓰는가?
- 스티커 메모가 화면을 과하게 가리지 않는가?
- 스크롤 영역이 자연스럽고 눈에 거슬리지 않는가?

### 신뢰감

- 앱이 너무 AI 생성물처럼 보이지 않는가?
- 로고와 색상이 골프 레슨/기록 서비스와 어울리는가?
- 프로가 회원에게 보여주기 부끄럽지 않은 완성도인가?
- 회원이 자신의 영상/사진이 어디에 저장되는지 불안하지 않은가?
- 개인정보/영상 보관 정책을 나중에 안내할 위치가 있는가?

## 5. 시연용 핵심 시나리오

### 시나리오 A: 일반회원 혼자 사용

1. 회원가입한다.
2. 홈에서 오늘 훈련을 확인한다.
3. 개인 훈련을 체크한다.
4. 영상 또는 이미지를 업로드한다.
5. 썸네일/스냅샷 기록을 확인한다.
6. 개인 기록 목록에서 다시 확인한다.

### 시나리오 B: 회원이 프로에게 연결

1. 회원이 프로를 검색한다.
2. 프로방 신청을 보낸다.
3. 프로가 신청을 승인한다.
4. 회원의 내 연결/프로방 목록이 갱신된다.
5. 회원이 기록을 프로방에 공유한다.
6. 프로가 공유 기록을 확인한다.

### 시나리오 C: 프로가 회원 관리

1. 프로가 회원을 검색한다.
2. 프로방 초대를 보낸다.
3. 회원이 초대를 수락한다.
4. 프로의 내 회원 목록이 갱신된다.
5. 프로가 개인 과제를 전송한다.
6. 회원이 과제를 완료한다.
7. 프로가 짧은 피드백을 남긴다.

### 시나리오 D: 영상 피드백

1. 회원이 스윙 영상을 올린다.
2. 프로가 영상을 0.5배속으로 확인한다.
3. 프로가 피드백 지점을 캡쳐한다.
4. 캡쳐 이미지 위에 본/각도 표시를 켠다.
5. 필요한 선/도형/텍스트 메모를 추가한다.
6. 불필요한 본/각도 일부를 지운다.
7. 회원에게 짧은 목표/주의점 피드백을 보낸다.

## 6. 판단 기준

- 통과: 사용자가 설명 없이도 다음 행동을 할 수 있다.
- 보완: 기능은 되지만 설명, 위치, 문구, 버튼 크기 중 하나가 헷갈린다.
- 실패: 사용자가 기능의 목적이나 다음 행동을 이해하지 못한다.

테스트 후에는 각 항목을 `통과 / 보완 / 실패 / 보류`로 표시한다.
