# GolfAlign MVP 테스트 체크리스트 v0.1

목적: 기능 수정 후 MVP가 회원, 프로, 관리자 기준으로 실제 사용 가능한지 확인한다.  
배포는 여러 수정 사항을 모은 뒤 마지막에 한 번에 진행한다.

## 1. 테스트 기준

- 로컬 테스트 URL: `http://localhost:3000`
- 운영 테스트 URL: `https://golf.aunova.ai`
- 로컬 빌드 확인: `npm.cmd run build`
- 운영 배포는 수정 사항을 모은 뒤 마지막에 진행한다.
- 테스트 계정과 관리자 계정은 외부 문서나 로그인 화면에 노출하지 않는다.

## 2. 로그인 / 세션

### 회원 / 프로 / 관리자 로그인

- 직접 회원가입 후 로그인한다.
- 회원으로 가입하면 회원 화면으로 이동한다.
- 프로로 가입하면 프로 화면으로 이동한다.
- 관리자 계정은 회원 화면과 프로 화면을 전환해 볼 수 있다.
- 마지막 로그인 아이디는 저장되지만 비밀번호는 저장되지 않는다.
- 다른 사람이 알 수 있는 테스트 계정 목록이 로그인 화면에 노출되지 않는다.

### 15분 무입력 자동 로그아웃

- 로그인 후 마우스, 키보드, 터치, 스크롤 입력이 있으면 타이머가 초기화된다.
- 15분 동안 입력이 없으면 자동 로그아웃된다.
- 자동 로그아웃 후 로그인 화면으로 돌아간다.
- 자동 로그아웃 안내 문구가 표시된다.
- 수동 로그아웃은 기존처럼 즉시 로그아웃된다.

## 3. 회원 기록 업로드

### 이미지 업로드

- 이미지 파일을 선택한다.
- 이미지 미리보기가 표시된다.
- 썸네일이 기록 목록에 표시된다.
- 기록 종류를 선택할 수 있다.
- 자세 분석 포인트를 선택할 수 있다.
- 개인 기록으로 저장된다.
- 프로방 공유를 켜면 프로방 공유 기록으로 저장된다.
- 프로 화면에서 공유된 이미지 기록이 보인다.

### 영상 업로드

- 영상 파일을 선택한다.
- 영상 길이가 확인된다.
- 5초 권장 안내가 표시된다.
- 7초 이하 영상은 자동 썸네일/스냅샷을 만든다.
- 7초 초과 영상은 MVP 정책에 따라 제한 또는 첫 구간 기준 스냅샷을 만든다.
- 기록 목록에 썸네일이 표시된다.
- 프로방 공유를 켜면 프로 화면에 피드백 대기 기록으로 표시된다.
- 프로 피드백 화면에서 원본 영상 보기 또는 다운로드 정책이 맞게 안내된다.
- 0.5배속 확인 UI가 표시된다.

### 저장 데이터 확인

- `records` 데이터에 `media`, `thumbnailUrl`, `mediaUrl`, `recordType`, `bodyAngle`, `roomId`, `roomName`이 저장된다.
- 개인 기록은 회원 본인에게만 보인다.
- 프로방 공유 기록은 연결된 프로에게 보인다.
- 다른 회원에게 영상/이미지 상세가 노출되지 않는다.

## 4. Google Drive / Google Sheets 저장

### Google Sheets

- 회원가입 정보가 `users`에 저장된다.
- 프로 정보가 `users`와 `pro_profiles`에 저장된다.
- 기록 메타데이터가 `records`에 저장된다.
- 프로방 공유 정보가 `shared_records`에 저장된다.
- 피드백 데이터가 `feedback`에 저장된다.
- 훈련 과제와 훈련 결과가 각각 저장된다.

### Google Drive

- 프로필 이미지는 MVP 기준 Google Drive 보관으로 준비한다.
- 프로필 이미지는 512x512 변환 후 저장하는 구조로 확장한다.
- 영상/이미지 원본은 앱이 장기 보관하지 않는 방향을 기본으로 한다.
- 썸네일, 캡처 이미지, 피드백 스냅샷은 Drive 저장 대상으로 본다.
- Drive 업로드 API 연결 후 `file_id`, `url`, `provider` 값이 DB에 남는지 확인한다.

## 5. NAS 연결 / 백업

MVP 기준 NAS:

- 모델: ipTIME NAS1 Dual
- 내부 IP: `192.168.0.102`
- 외부 주소: `https://novart.ipdisk.co.kr`

### 연결 테스트

- 같은 네트워크에서 `192.168.0.102` 접속 가능 여부를 확인한다.
- 외부 주소 `https://novart.ipdisk.co.kr` 접속 가능 여부를 확인한다.
- WebDAV, SFTP, NAS API 중 사용할 연결 방식을 확정한다.
- NAS 인증 정보는 코드에 직접 저장하지 않는다.
- NAS 접속 실패 시 앱 사용이 막히지 않아야 한다.

### 백업 테스트

- Google Drive에 저장된 테스트 파일을 NAS로 복사한다.
- 파일명은 `record_id` 또는 `snapshot_id` 기준으로 저장한다.
- NAS 경로 예시:

```text
/GolfAlign/
  users/
    {user_id}/
      records/
      thumbnails/
      snapshots/
  rooms/
    {room_id}/
      shared_records/
      feedback/
  backups/
    sheets/
```

- 백업 성공 시 `backup_provider=nas`, `backup_path`, `sync_status=done`을 기록한다.
- 백업 실패 시 `sync_status=failed`, `error_message`를 기록한다.
- 백업 실패 기록은 관리자 화면에서 확인할 수 있어야 한다.

### 영상 아카이브 정책 테스트
- 영상 업로드 시 `archive_policy=feedback_done_plus_7_or_upload_plus_14`가 저장된다.
- 영상 업로드 시 `archive_after_days=7`이 저장된다.
- 영상 업로드 시 `archive_due_at`은 업로드 시점 기준 14일 뒤로 저장된다.
- 이미지 업로드 시 `archive_policy=none`으로 저장된다.
- 피드백 완료 후 7일이 지난 영상은 NAS 이동 후보로 잡힌다.
- 피드백이 완료되지 않아도 업로드 후 14일이 지난 영상은 NAS 이동 후보로 잡힌다.
- NAS 이동 성공 전에는 Google Drive/임시 원본을 삭제하지 않는다.
- NAS 이동 성공 시 `backup_provider=nas`, `backup_path`, `sync_status=backed_up`, `archived_at`을 기록한다.
- NAS 이동 실패 시 `sync_status=failed`로 남기고 관리자 지표에서 확인한다.

## 6. 프로방 / 메시지

- 회원이 프로를 검색한다.
- 회원이 프로방 신청을 보낸다.
- 프로가 회원을 검색한다.
- 프로가 회원에게 초대 메시지를 보낸다.
- 회원이 초대를 수락한다.
- 프로가 신청을 승인한다.
- 승인 후 회원의 내 연결 목록이 갱신된다.
- 승인 후 프로의 내 회원 목록이 갱신된다.
- 회원과 프로가 개인 메시지를 주고받는다.
- 메시지가 많아져도 메시지함 스크롤이 자연스럽게 동작한다.

## 7. 훈련

- 회원의 방 공통 드릴 체크가 완료/진행 중으로 토글된다.
- 회원의 개인 훈련 체크가 완료/대기로 토글된다.
- 회원이 훈련 결과를 기록한다.
- 결과에 횟수, 난이도, 메모를 남긴다.
- 영상/이미지 첨부는 선택으로 둔다.
- 프로가 회원의 훈련 결과를 확인한다.
- 프로가 짧은 코멘트를 남긴다.
- 프로 코멘트가 회원에게 보인다.

## 8. 프로 피드백

- 프로가 공유된 회원 기록을 연다.
- 영상이면 0.5배속 확인 UI가 보인다.
- 현재 장면 캡처가 가능하다.
- 캡처 이미지 위에 텍스트 메모를 남긴다.
- 텍스트 색상을 선택할 수 있다.
- 텍스트를 이동할 수 있다.
- 선, 화살표, 사각형, 원을 그릴 수 있다.
- 지우개로 메모나 도형을 제거할 수 있다.
- 뼈각도 레이어를 켜고 끌 수 있다.
- 특정 뼈라인이나 각도만 숨길 수 있다.
- 피드백 저장 후 회원 상세 화면에 반영된다.

## 9. 관리자 지표 / 사용량 관리 의견

관리자 화면에는 MVP 후반부터 운영 지표 대시보드를 두는 것이 좋다.  
처음부터 모든 분석 화면을 만들 필요는 없지만, 데이터는 지금부터 쌓을 수 있게 설계해야 한다.

### 관리자에게 필요한 지표

- 전체 회원 수
- 전체 프로 수
- 활성 회원 수: 최근 1일, 7일, 30일
- 활성 프로 수: 최근 1일, 7일, 30일
- 신규 가입 수
- 로그인 횟수
- 영상 업로드 수
- 이미지 업로드 수
- 피드백 작성 수
- 훈련 과제 발송 수
- 훈련 결과 제출 수
- 메시지 발송 수
- 프로방 신청 수
- 프로방 초대 수
- 프로방 승인 수

### 저장 용량 지표

- 전체 저장 용량
- 사용자별 저장 용량
- 프로방별 저장 용량
- 파일 종류별 저장 용량: 영상, 이미지, 썸네일, 스냅샷, 프로필 이미지
- 저장소별 저장 용량: Google Drive, NAS, local, future storage
- 백업 성공/실패 수
- 백업 대기 파일 수
- 30일 미접속 회원의 프로필 이미지/NAS 이동 후보 수

### 파트별 사용량

- 기록 업로드 사용량
- 프로 피드백 사용량
- 훈련 과제 사용량
- 메시지 사용량
- 프로방 신청/초대 사용량
- 로그인/가입 사용량
- 스냅샷/뼈각도 분석 사용량

### 추천 데이터 구조

초기에는 Google Sheets에 아래 시트를 추가하는 방식이 적합하다.

- `usage_events`: 사용 이벤트 원본 로그
- `storage_usage`: 파일/저장소별 용량 로그
- `backup_jobs`: NAS 백업 작업 상태
- `admin_daily_metrics`: 일별 집계 지표

`usage_events` 예시 필드:

| 필드 | 설명 |
|---|---|
| id | 이벤트 ID |
| user_id | 사용자 ID |
| role | member, pro, admin |
| event_type | login, upload_video, upload_image, feedback_save 등 |
| target_type | record, feedback, training, message 등 |
| target_id | 대상 ID |
| room_id | 프로방 ID |
| file_size_bytes | 파일 용량 |
| storage_provider | drive, nas, local |
| created_at | 발생 시각 |

`storage_usage` 예시 필드:

| 필드 | 설명 |
|---|---|
| id | 저장 로그 ID |
| owner_user_id | 소유자 |
| room_id | 연결 프로방 |
| record_id | 기록 ID |
| file_kind | original_video, image, thumbnail, snapshot, profile_image |
| provider | drive, nas, local |
| file_id | 저장소 파일 ID |
| path_or_url | 경로 또는 URL |
| size_bytes | 용량 |
| backup_status | pending, done, failed |
| created_at | 저장 시각 |

### 관리자 화면 MVP 범위

처음부터 복잡한 차트보다 아래 카드형 지표로 시작한다.

- 오늘 가입
- 오늘 업로드
- 오늘 피드백
- 오늘 훈련 제출
- 이번 달 총 저장 용량
- NAS 백업 실패
- Google Drive 사용량
- 최근 오류 로그
